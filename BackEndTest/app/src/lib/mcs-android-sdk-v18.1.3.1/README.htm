Oracle Mobile Cloud Service (MCS) Android SDK 18.1.3.1

Requirements

MCS Android SDK version 18.1.3.1 is compatible with Android 4.4 (API Level 19) to Android 8.1 (API Level 27).

Contents

The mcs-android-sdk-18.1.3.1.zip contains the following SDK libraries:

  • mcs-android-sdk-shared-v18.1.3.1*.jar is the base library for the SDK, including functionality required by the other libraries as well as utility classes for accessing and authenticating with mobile backends.

  • mcs-android-sdk-analytics-v18.1.3.1*.jar is the Analytics library, which lets you insert custom events into your code that can then be collected and analyzed from the Analytics console.

  • mcs-android-sdk-location-v18.1.3.1*.jar is the Location library, which lets you access details about location devices that have been registered in MCS and their associated places and assets.

  • mcs-android-sdk-fcm-notifications-v18.1.3.1*.jar is the Notifications library for FCM, which lets you set up your application to receive notifications sent from your mobile backend. If your app still uses GCM, the SDK also includes /gcm/mcs-android-sdk-notifications-v18.1.3.1*.jar. (The two notifications modules canâ€™t be used at the same time.)

  • mcs-android-sdk-social-v18.1.3.1*.jar is the Social Login library, which allows you to set up your app to use Facebook login.

  • mcs-android-sdk-storage-v18.1.3.1*.jar is the Storage library, which lets you write code to access storage collections that are set up with your mobile backend.

  • mcs-android-sdk-sync-v18.1.3.1*.jar is the Sync Client library, which allows you to cache application data when the device running your app is disconnected from the network, then sync up the data when the network connection is reestablished.

  • IDMMobileSDK.jar is the identity management library used by all applications.

The SDK package includes the complete SDK API documentation in Javadoc.zip. You can also reference the API documentation online: https://docs.oracle.com/en/cloud/paas/mobile-cloud/mcssa/index.html.

Tools

In addition to its libraries, the SDK includes useful development tools:

  • mcs-tools.zip is a collection of command-line development tools, including a container to run and debug your APIs locally, a testing tool to submit requests to your API implementation, and a deployment tool to package your implementation and deploy it to MCS.

  • mobile-log-download.zip is a command-line tool that allows you to download logs from MCS for viewing or archiving.

Samples

The SDK package also includes the following samples:

  • examples.zip contains sample mobile apps that demonstrate how to use the SDK.

  • oracle_mobile_cloud_config.xml is a sample configuration file. You can adjust its properties based on the environment details of the mobile backend that your app will use and then copy the file to the assets folder you created when adding the SDK to your app.
